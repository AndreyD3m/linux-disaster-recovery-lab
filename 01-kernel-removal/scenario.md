# Сценарий: Удаление всех ядер Linux

## Цель
Довести систему до состояния, когда она не может загрузиться (kernel panic / memtest), и восстановить её через LiveCD.

## Окружение
- Виртуальная машина: VirtualBox 7.1
- ОС: Ubuntu 20.04 (focal)
- RAM: 2 ГБ
- Диск: 20 ГБ

## Шаги воспроизведения (поломка)

1. Узнать текущее ядро:
   uname -r
2. Важно! У нас должно быть минимум 2 ядра, если только 1 – надо установить второе
    apt update && apt install linux-image-6.17.0-10-generic -y
3. Если команда не прошла и ядро не установилось, можно попробовать способ который подошел мне  
	1. Добавьте следующую строку в файл `/etc/apt/sources.list`  
		deb http://archive.ubuntu.com/ubuntu/ questing-proposed main     
	2. Обновите индекс пакетов:
	    sudo apt-get update
	3. Установите пакет linux-image-6.17.0-10-generic в формате deb:
	    sudo apt-get install linux-image-6.17.0-10-generic
4. Приступаем к удалению ядра системы 
   apt remove --purge linux-image-6.14.0-33-generic -y
   Появится окно с предупреждением о том что мы удаляем ядро на котором в данный момент стоит система, мы нажимаем нет, так как хотим продолжить процесс удаления.  
5. Если не удалось удалить ядро данной командой, то можно запустить принудительное удаление 
   dpkg --remove linux-image-6.14.0-33-generic
   1. Если ядро имеет зависимости, то команда не сработает, перед этим надо их удалить
        apt remove --purge linux-image-generic-hwe-24.04 -y
После удаления перезапускаем систему и видим черный экран или Memtest86, так как системе надо что-то показать
