# Сценарий: Заполнение диска (disk full)

## Цель
Заполнить диск до отказа, чтобы система перестала принимать новые файлы, и восстановить работоспособность, найдя и удалив «зомби-файлы».

## Окружение
- Виртуальная машина: VirtualBox 7.1
- ОС: Ubuntu 20.04 (focal)
- RAM: 2 ГБ
- Диск: 20 ГБ (свободно ~14 ГБ)

## Инструменты
- `dd` — создание файла-гиганта
- `df` — мониторинг свободного места
- `du` — поиск больших файлов
- `lsof` — поиск открытых удалённых файлов

## Шаги воспроизведения (поломка)

1. Проверить текущее состояние диска
    ```bash
    df -h
Запомнить, сколько свободно (например, 14 ГБ).

2. Заполнить диск большим файлом
Создадим файл размером чуть больше свободного места:
    ```bash
    dd if=/dev/zero of=~/bigfile bs=1G count=15
Если места меньше, то надо изменить count.
Через некоторое время появится ошибка:
    ```text
    dd: writing to '~/bigfile': No space left on device
3. Убедиться, что диск заполнен
    ```bash
    df -h
Avail должен быть 0, Use% — 100%.
