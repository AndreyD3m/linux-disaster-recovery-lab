# Решение: Анализ работы OOM Killer

## Что произошло
При нехватке памяти ядро Linux активирует механизм OOM Killer, который выбирает и завершает процесс, чтобы предотвратить крах всей системы.

## Диагностика

1. Проверка, сработал ли OOM Killer
    ```bash
    dmesg | grep -i "oom-killer"
        Должны быть строки:
        ```text
        [ 1234.567890] stress-ng invoked oom-killer: ...
        [ 1234.567912] Killed process 1234 (stress-ng)
2. Анализ состояния памяти до и после
    ```bash
   free -h
До: available близко к нулю
После: память освободилась

3. Принудительно завершаем процесс с помощью htop(если OOM не сработал)
    1. Выделяем нужный процесс стрелками
    2. Нажми F9 (или клавишу k) 
    3. Появится меню выбора сигнала:
        По умолчанию выбран SIGTERM (15) — вежливое завершение 
        Если процесс не отвечает, выбери SIGKILL (9) — принудительное убийство 
    4. Нажми Enter для подтверждения
# Почему OOM Killer может НЕ сработать
Причина	Описание
Фрагментация	Свободной памяти много, но нет непрерывного куска
Лимиты cgroups	Ограничения внутри контейнера
Слишком маленькая нагрузка	mmap без записи не жрёт физическую память
