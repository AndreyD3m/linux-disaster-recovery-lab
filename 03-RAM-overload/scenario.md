## Сценарий: Перегрузка оперативной памяти (RAM overload)

## Цель
Создать критическую нагрузку на оперативную память, чтобы вызвать срабатывание OOM Killer и принудительное завершение процессов.

## Окружение
- Виртуальная машина: VirtualBox 7.1
- ОС: Ubuntu 20.04 / 22.04
- RAM: 2–4 ГБ (специально ограничено)
- Swap: отключён для чистоты эксперимента

## Инструменты
- `stress-ng` — утилита для создания нагрузки
- `htop` / `vmstat` — мониторинг
- `dmesg` — просмотр логов ядра

## Шаги воспроизведения

1. Установить stress-ng (если не установлен)
    ```bash
    sudo apt update
    sudo apt install stress-ng -y
2. Отключить swap (чтобы OOM сработал быстрее)
    ```bash
    sudo swapoff -a
3. Запустить мониторинг в отдельном терминале
    ```bash
    htop # Терминал 1: наблюдаем за памятью
    ```bash
    sudo dmesg -w # Терминал 2: смотрим логи ядра в реальном времени
4. Запустить пожирателя памяти
    ```bash
    stress-ng --vm 3 --vm-bytes 1G --vm-keep # Пример для машины с 2 ГБ RAM
